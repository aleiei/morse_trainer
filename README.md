# Morse Trainer

This project was created from a real practical need: a simple and usable tool for people who want to learn CW.

It was built using an Arduino Nano and an ST7920 128x64 display because those parts were already available in my lab.

The whole project is designed around component reuse, with the goal of building a useful CW trainer from hardware already on hand.

## Project goals

- Provide a practical CW training tool for daily use
- Keep the hardware simple and accessible
- Reuse existing components in the lab

## Hardware used

- Arduino Nano (ATmega328P)
- ST7920 128x64 LCD (hardware SPI)
- Rotary encoder with push button
- Buzzer
- Vertical key and optional iambic paddle

## Libraries used

- U8g2
	- External library for ST7920 LCD rendering.
	- Installable from PlatformIO/Library Manager.
- SPI
	- Arduino core library used for hardware SPI communication with the display.
- EEPROM
	- Arduino core library used to store persistent settings (key type, speed mode, WPM).

## Development environment

This project was written in Visual Studio Code and built with PlatformIO.

- Primary workflow
	- Editor: Visual Studio Code
	- Build/upload: PlatformIO
	- Main source file: `src/morse_trainer.cpp`

## Using Arduino IDE

The project can also be used with Arduino IDE.

- Required changes
	- Create an Arduino sketch folder named `morse_trainer`.
	- Copy `src/morse_trainer.cpp` into that folder.
	- Rename the copied file to `morse_trainer.ino`.

- Arduino IDE setup
	- Board: Arduino Nano
	- Processor: ATmega328P (select the correct bootloader for your board)
	- Port: select your serial port
	- Install library: U8g2 (Library Manager)

- Notes
	- `SPI` and `EEPROM` are included in the Arduino core and do not need manual installation.
	- Pin mapping and project behavior are the same as in the PlatformIO version.

## Menu and functions

The user interface is controlled by a rotary encoder with push button.

- From the training screen, press the encoder to open the menu.
- Rotate the encoder to move through menu items.
- Press the encoder to confirm a selection.

Root menu entries:

- Key type
	- Vertical
	- Iambic (2 lever)
- Speed
	- Manual
	- Automatic (available for vertical key mode)
- Clear screen
	- Clears the decoded text buffer shown in the training area
- Back
	- Returns to the training screen

Speed behavior:

- Manual mode
	- The WPM value is set directly by the user.
	- Selecting Manual opens the WPM editor.
- Automatic mode
	- The trainer estimates WPM from recent timing samples.
	- Auto estimation is used in vertical key mode.
- Iambic mode
	- Timing is generated by the keyer logic.
	- Speed mode is kept in Manual for consistent operation.

WPM editor:

- Rotate encoder to increase or decrease WPM.
- Press encoder to save and return to the menu.

Training screen:

- Displays key type and current speed.
- Shows decoded characters in the text area.
- Adds letter/word separation according to Morse timing gaps.
